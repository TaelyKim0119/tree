# 사주명리 웹 앱 설계 문서

## 개요

생년월일시를 입력하면 만세력을 계산하고, Gemini API를 활용해 종합 사주 풀이를 제공하는 웹 앱.

## 핵심 요구사항

- **입력**: 성별, 양력/음력, 생년월일, 태어난 시간
- **만세력 계산**: 규칙 기반으로 년주/월주/일주/시주 계산
- **사주 풀이**: Gemini API에 만세력 JSON + 전문가 시스템 프롬프트 전달 → 자연어 풀이 생성
- **풀이 범위**: 종합 (12단계 분석)
- **인증**: 없음 (바로 사용)
- **반응형**: 모바일/태블릿/데스크톱 지원

### 화면 구성 (탭/멀티페이지)

#### 1. 메인 화면 (HomePage)
- 풀스크린 배경 이미지 (보석 나선) + 오버레이
- 오행 5원소 원형 아이콘 (플로팅 애니메이션)
- "사주명리" 타이틀 (골드 그라데이션 텍스트)
- 두 개의 글래스모피즘 메뉴 카드:
  - "내 사주 보기" (큰 나무 아이콘 + 들판 배경)
  - "궁합 보기" (컬러 돌 아이콘 + 호수 배경)

#### 2. 입력 화면 (SajuInputPage)
- 배경: 폭포 이미지 + 오버레이
- 오행 5장 이미지 스트립 배너
- 입력 폼:
  - 성별 (남/여 토글)
  - 양력/음력 토글
  - 생년월일 (년/월/일 드롭다운)
  - 태어난 시간 (시진 선택)
- CTA 버튼 (골드 그라데이션 + 시머 효과)

#### 3. 궁합 입력 화면 (GunghapInputPage)
- 두 사람의 사주 정보 입력
- 좌/우 분할 레이아웃

#### 4. 결과 화면 (SajuResultPage)
- **사주팔자 히어로**: 보석 배경 + 4주 카드 (각 칸에 오행 이미지)
  - 일주 강조 (amber 링 + 日 뱃지)
  - 신강/신약, 용신, 격국 뱃지
- **10개 탭**:
  - 원국분석 (1~2단계)
  - 성격 (장점/단점)
  - 금전운 (4단계)
  - 직업운 (5단계)
  - 연애운 (6단계)
  - 결혼운 (7단계)
  - 건강운 (8단계)
  - 가족/관계 (9단계)
  - 대운/세운 (10~11단계)
  - 총평 (3+12단계)

#### 5. 궁합 결과 화면 (GunghapResultPage)
- 두 사람의 사주 비교
- 궁합 종합 점수 및 분석

## 만세력 계산 (서버 규칙 기반)

| 계산 항목 | 방식 |
|----------|------|
| 년주 | 생년 기준 60갑자 순환 (절입 기준 적용) |
| 월주 | 생월 + 년간 기준 (월건법) |
| 일주 | 기준일(1900-01-01 = 甲子)로부터 일수 차 |
| 시주 | 생시 + 일간 기준 (시건법) |
| 오행 분포 | 8글자의 목/화/토/금/수 비율 |
| 십신 | 일간 기준 나머지 7글자와의 관계 |
| 신강/신약 | 월지 득령, 인성/비겁 개수 판단 |
| 용신 | 오행 균형 분석으로 부족한 기운 판별 |
| 대운 | 월주 기준 순행/역행 10년 단위 |
| 세운 | 해당 년도의 천간/지지 |

## Gemini API 연동

### 시스템 프롬프트
`제미나이 사주 전문가 지침.txt` 내용을 기반으로 하되, "캡처본" 관련 문구를 "아래 만세력 데이터" 기반으로 수정.

### 요청 구조
```json
{
  "시스템프롬프트": "30년 경력 전문 역술가 지침...",
  "사용자메시지": {
    "성별": "남",
    "양력생일": "1990-03-15",
    "출생시간": "14:30",
    "년주": { "천간": "庚", "지지": "午" },
    "월주": { "천간": "己", "지지": "卯" },
    "일주": { "천간": "甲", "지지": "子" },
    "시주": { "천간": "辛", "지지": "未" },
    "오행분포": { "목": 3, "화": 1, "토": 2, "금": 1, "수": 1 },
    "십신": { ... },
    "신강신약": "신강",
    "용신": "水",
    "대운": [ ... ],
    "세운": [ ... ]
  }
}
```

### 응답 파싱
Gemini 응답을 12단계 섹션 헤더로 파싱 → 각 탭에 매핑하여 표시.

## API 엔드포인트

### POST /api/saju
- 요청: `{ gender, calendarType, year, month, day, hour }`
- 응답: `{ manseryeok: {...}, analysis: "..." }`

### POST /api/gunghap
- 요청: `{ person1: {...}, person2: {...} }`
- 응답: `{ manseryeok1: {...}, manseryeok2: {...}, analysis: "..." }`

## 에러 처리
- 유효하지 않은 날짜 → 프론트에서 검증
- Gemini API 실패 → 재시도 1회 + 에러 메시지 표시
- 만세력 계산 범위 초과 (1900~2100) → 안내 메시지

## 성능 고려사항
- Gemini API 응답이 길어질 수 있으므로 스트리밍 응답 적용
- 이미지 lazy loading + WebP 변환 고려
- 결과 화면 탭 전환은 클라이언트 사이드
